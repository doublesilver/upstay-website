## Q5. 워터마크 박힌 기존 이미지 처리 (CRITICAL)

### 진단 결과 (2026-04-28)

`data/upstay.db` SQL 진단:

```
SELECT COUNT(*) FROM case_images WHERE image_url != ''
→ 25개

SELECT COUNT(*) WHERE image_url_wm != '' AND image_url_wm = image_url
→ 0개

SELECT COUNT(*) WHERE image_url_wm != '' AND image_url_wm != image_url
→ 0개

SELECT COUNT(*) WHERE image_url_wm = ''
→ 25개
```

**결론**: 현재 production DB에 `image_url_wm` (별도 워터마크 적용본) 컬럼이 채워진 이미지가 **0개**. 즉 모든 이미지가 `image_url`만 사용 중이며, **"11시 방향 안 지워지는 로고"는 `image_url` 자체에 워터마크가 영구 합성되어 업로드된 상태**로 추정.

### 코드 동작 검증

- 신규 시스템(`image_url` 원본 + `image_url_wm` 워터마크 적용본 분리 저장)은 코드 레벨에 **이미 구현되어 있음** (`lib/db/index.ts:46`, `components/admin/image-edit-modal.tsx:313-328`)
- 폴백 로직: `image_url_wm || image_url` (`lib/home-data.ts:56`, `app/remodeling/[id]/page.tsx:56,60`) — 변경 없음 (안전)
- 향후 워터마크 적용 시 `image_url`은 보존되고 `image_url_wm`에만 합성본 저장됨

## Q6. 사례 상세 화면 "설명글 복구" 정확한 의도

대표님 메시지: _"두번째 화면이 세로로 너무 길어지고 설명글이 없어졌습니다"_

### 현재 진단

- `remodeling_cases` 테이블에 `description` 컬럼 **없음** (title만 존재)
- admin "내용" 라벨 textarea가 사실상 `title` 컬럼에 저장 중 → v3.5에서 라벨을 "설명"으로 통일
- 모바일 상세 페이지 좌측 회색 라벨 + 본문 박스에 title 렌더 (v3.5에서 디자인 정리)
- PC 상세 페이지에는 갤러리만 표시 (viewport 100% 사용) — 설명 박스 없음

### 미확인

- v2 → v3 전환에서 사라진 "설명글"이 정확히:
  - (a) **모바일 설명 박스**가 사라졌었는지 — 현재 있음, 디자인만 v3.5 사양으로 변경
  - (b) **PC 설명 박스**가 필요한지 — 현재 없음, 갤러리가 viewport 100%
  - (c) **별도 description 필드**가 사라졌는지 — DB 컬럼 자체가 없으므로 해당 없음
- "세로로 너무 길어진다"는 모바일 기준인지, PC 기준인지
