# 클라이언트 확인 필요 사항

## Q1. 사진안내 카테고리의 정확한 범위
- 현재 해석: 메인 페이지의 `리모델링 사례보기 →` 영역 텍스트/노출 제어
- 궁금한 점: 이 기능이 사례보기 버튼 영역 관리인지, 별도 신규 섹션인지
- 현재 구현 상태: 메인 상단 사례보기 버튼/보조문구 제어용으로 임시 구현

## Q2. 별표 4개의 실제 노출 방식
- 현재 해석: BEFORE 최대 4장, AFTER 최대 4장을 별표로 선택하고 메인/상세에 별표된 이미지만 순서대로 노출
- 궁금한 점: 메인에서 4장 갤러리인지, 대표 1장과 보조 썸네일 조합인지
- 현재 구현 상태: 4장 갤러리 기준으로 임시 구현

## Q3. 워터마크 두께의 의미
- 현재 해석: 워터마크 로고 주변 그림자/아웃라인 강도
- 대안 해석: 로고 자체 선 굵기 변경
- 현재 구현 상태: 그림자 방식으로 임시 구현

## Q4. 메인 페이지 `(Before → After)` 문구 의도
- 현재 해석: 기존 문구 유지, 크기와 회색 톤만 소폭 정리
- 궁금한 점: 문구 자체 변경이 필요한지, 사진안내 카테고리와 연결되는지
- 현재 구현 상태: 기존 문구 유지, 크기와 회색 톤만 소폭 조정
